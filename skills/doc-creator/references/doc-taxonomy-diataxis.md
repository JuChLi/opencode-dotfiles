# 文件分類（Diataxis）

本 skill 使用 Diataxis 的四象限分類來決定文件目的與結構，避免把不同任務混在同一份文件。

## 四種類型

| 類型 | 核心問題 | 讀者預期 | 常見輸出 |
|---|---|---|---|
| `tutorial` | 我該怎麼學會？ | 跟著做就能成功 | 入門教學、快速開始 |
| `how-to` | 我該怎麼完成一件事？ | 快速解決當前任務 | 操作指南、排障流程 |
| `reference` | 系統規格是什麼？ | 精準查詢、可引用 | API/參數/欄位規格 |
| `explanation` | 為什麼這樣設計？ | 理解背景與取捨 | 設計說明、架構決策 |

## 快速判斷

1. 目標是「學習」還是「完成任務」？
   - 學習 -> `tutorial`
   - 完成任務 -> `how-to`
2. 目標是「查規格」還是「懂原因」？
   - 查規格 -> `reference`
   - 懂原因 -> `explanation`

## 交接文件（`handoff`）

`handoff` 是本 skill 額外補充的實務類型，目的是讓交接者在最短時間內接手：

- 先看 `TL;DR` 確認現況。
- 再看 `待辦事項`、`風險與阻塞`、`回滾與應變`。
- 最後依 `驗證與結果` 重現現況。

## 分類錯誤的常見症狀

- `tutorial` 直接堆滿規格表，缺乏可完成任務的步驟。
- `how-to` 變成背景長文，缺少可執行的動作。
- `reference` 出現大量「建議/故事」，導致難以查詢。
- `explanation` 只有步驟，沒有決策依據與取捨。
