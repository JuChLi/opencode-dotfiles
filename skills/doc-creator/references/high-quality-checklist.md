# 高品質技術文件檢查清單

提交前請逐項確認：

## 結構

- [ ] 已明確標示文件類型（tutorial/how-to/reference/explanation/handoff）。
- [ ] 章節完整且符合對應模板。
- [ ] 標題具描述性，不使用「其他」「雜項」「備註」等籠統標題。

## 可讀性

- [ ] 每段聚焦單一重點，避免長段落。
- [ ] 操作步驟具可執行性，非抽象敘述。
- [ ] 範例可直接複製使用（含必要前置條件）。

## 術語一致性

- [ ] 同一概念只使用一種術語。
- [ ] 「回傳/逾時/流程」等核心詞彙在整份文件一致。
- [ ] 禁用模板句與佔位字（如 `TODO`、`待補`）已移除。

## 內容正確性

- [ ] 文件內容與現行程式/流程一致。
- [ ] 限制、風險、回滾方式有明確記錄。
- [ ] 若有版本差異，已標示適用版本。

## 交接友善（handoff 類型必檢）

- [ ] `TL;DR` 能在 1 分鐘內說明當前狀態。
- [ ] `待辦事項` 有明確優先序與下一步。
- [ ] `風險與阻塞` 包含觸發條件與應對方案。
- [ ] `驗證與結果` 可讓接手者重現現況。

## 品質閘門

- [ ] `validate_structure.py` 通過。
- [ ] `check_terminology.py` 通過。
- [ ] `lint_prose_zhtw.py` 已檢視 warning 並合理修正。
- [ ] `detect_similarity.py` 已確認無不必要重複文件。
