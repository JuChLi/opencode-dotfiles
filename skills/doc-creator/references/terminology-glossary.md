# 文件術語表（zh-TW）

本文件用於統一 `doc-creator` 的常用詞彙，降低跨文件的語意漂移。

## 核心術語

| 建議用語 | 避免用語 | 說明 |
|---|---|---|
| 回傳 | 返回 | 描述函式或流程輸出時統一使用。 |
| 逾時 | 超時 | 描述 timeout 情境時統一使用。 |
| 先備條件 | 前置條件（作為章節名稱時） | 文件模板章節名稱統一。 |
| 交接 | 移交（作為章節名稱時） | 交接文件固定章節與術語。 |

## 文件類型術語

- `tutorial`：教學型文件。
- `how-to`：任務導向操作文件。
- `reference`：規格查詢文件。
- `explanation`：設計與原理解釋文件。
- `handoff`：交接文件。

## 品質流程術語

- 結構檢查（structure validation）
- 術語檢查（terminology check）
- 文風 lint（prose lint）
- 相似度偵測（similarity detection）
- 品質閘門（quality gate）

## 建議語氣

- 使用台灣繁體中文。
- 優先使用直接、可執行、可驗證的句型。
- 避免低資訊句型（例如「執行某功能」「如上所述」「待補」）。
