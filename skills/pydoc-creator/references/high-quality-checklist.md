# 高品質 Python Docstring 檢查清單

提交前請逐項確認：

## 覆蓋率

- [ ] 所有公開模組都有 module docstring。
- [ ] 所有公開類別都有 class docstring。
- [ ] 所有公開函式/方法都有 docstring。
- [ ] 若專案需要，private API 也有一致標準。

## 語意正確性

- [ ] 摘要首句與實際行為一致。
- [ ] 參數說明描述用途，不是重複參數名稱。
- [ ] 回傳說明反映真實回傳語意。
- [ ] 例外說明反映真實拋出條件。
- [ ] 摘要後有至少一段詳細描述（用途、限制、注意事項）。

## Python 語意

- [ ] `async def` 有描述非同步完成語意。
- [ ] 生成器（`yield`）有說明產生值語意。
- [ ] 不誤導副作用（I/O、外部呼叫、狀態變更）。

## 語言品質（zh-TW）

- [ ] 使用台灣繁體中文。
- [ ] 不使用 emoji 與裝飾符號。
- [ ] 避免模板句（例如「執行某功能」「輸入參數」）。

## 一致性

- [ ] 同類型函式（`get/is/set`）描述風格一致。
- [ ] 同一模組術語一致。
- [ ] 文件更新與程式碼更新同步。
- [ ] Google 風格下，`Args/Returns/Raises/Examples` 區段命名一致。

## 風格可追溯性

- [ ] 已記錄目前使用的風格設定檔（`pep257/google/custom`）。
- [ ] 若使用 custom style，已保存來源 URL 或文件路徑。
- [ ] custom style 規則可由腳本重現。
